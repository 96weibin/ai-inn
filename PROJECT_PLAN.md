## 📋 项目概述

AI-Inn 是一款面向民宿行业的智能管理系统，核心特点是通过 AI 助手实现自然语言交互，让房东可以随时随地通过语音或文字指令完成房间管理、入住办理等操作。

### 核心价值

- 🤖 **AI 智能助手** - 自然语言交互，随时随地管理民宿
- 📊 **多视图展示** - 表格、楼层图、3D视图等多种展示方式
- 🌐 **多平台对接** - 支持 Booking、Airbnb、美团等平台订单
- 💰 **智能排房** - 自动分析最优房间安排方案
- 📱 **移动友好** - 支持手机端操作

### 2026-05-19（开发进展 - 命名体系重构）

**完成内容**：
- ✅ **全局命名统一：`roomType` → `roomTemplate` 全链路重构
  - TypeScript 类型名：`RoomTypeItem` → `RoomTemplate`
  - API 对象：`roomTypeApi` → `roomTemplateApi`
  - 文件名：`RoomTypeManagement.tsx` → `RoomTemplateManagement.tsx`
  - 路由引用 App.tsx 同步更新
  - 变量命名：`roomTypes` → `templates`
- ✅ **接口路由统一：`/api/room_templates` (snake_case)
- ✅ **双层 ID 模式确定**：
  - 数据库内部自增 `id`（JOINs）
  - API/前端外部通信用 `uid`（UUID）
- ✅ **房间创建选房型时提示 `type is required` Bug 彻底修复
  - 后端房间表改用 `room_template_uid` 关联模板表
  - 前端下拉 Option 的 key/value 全程用 uid
  - 表单验证消息从 "type is required" 改成 "请选择房型"
- ✅ **删除确认流程全链路覆盖
  - 房间配置删除房间前 Modal.confirm
  - 房型管理删除前 Modal.confirm
  - 避免 Dropdown -> Menu 事件冲突解决（改成 imperative 方式）
- ✅ ** rooms.py 后端升级：接收 room_template_uid 查询反查模板表回填 type/price/guests 等默认值
- ✅ 列表渲染：兼容 uid优先匹配 room_template_uid，降级匹配旧数据仍然兼容 type **TypeScript 类型检查 0 Error

**技术决策：**
1. **命名一致性高于一切
2. **房间与模板表关系图：
   ```
   RoomTemplate { uid  ----------+
                              |
                              | uid (关联）
                              ↓                           
   Room (room_template_uid) ---(通过反查填充数据
   ```

### 2026-05-19 第二轮（下午迭代：全程 uid-only 架构 + 预订产品化流程）

**核心重构完成内容**：
- ✅ **全程 uid-only 架构，内部自增 id 永不外露**
  - `types/index.ts` 定义层：Room/Guest/Booking 三大实体 `id` 字段完全删除，只保留 `uid: string`
  - API 签名层：CRUD 参数全部 `(uid: string)`，没有任何方法继续接收 `id: number`
  - 后端序列化层：rooms/guests/bookings 四个列表 GET 方法全不返回 `id` 整数
  - 路由层通配：全部资源路由 `/api/rooms/{room_uid}` `/api/guests/{guest_uid}`
  - 六个页面：全部 Table.rowKey="uid"，所有更新删除提交 uid
- ✅ **预订管理产品化流程大重构** [BookingManagement.tsx](file:///d:/github/ai-inn/src/pages/BookingManagement.tsx#L277-L300)
  - **STEP:1 先选日期**：RangePicker 移到表单顶部第一字段
  - **STEP:2 动态房态**：选完日期自动计算该时间段可售库存，时间线重叠检测算法：已有预订.start < 新预订.end AND 已有预订.end > 新预订.start
  - **STEP:3 房晚统计**：右边实时显示蓝 Tag「共N+1天」绿 Tag「N晚」
  - 房间 Option 标签美化：房号 + 房型彩色 Tag + 价格 + 清洁/维护状态
  - 客人 Option 识别老客人：姓名 + 手机 + `回头客N次` 绿标
- ✅ 删除确认统一 Modal.confirm() 方案，避免嵌套 Popconfirm 事件冒泡冲突
- ✅ **TypeScript 0 Errors 持续保持，全链路类型定义签名一致**

**用户体验对比**：

| 模块 | 之前的问题 | 现在的体验 |
|-----|-----------|-----------|
